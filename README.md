# HL Terminal

[Hyperliquid](https://hyperliquid.xyz) の自作 Web フロントエンド。公開 API（REST + WebSocket）を
ブラウザから直接叩く、ビルド不要の静的ファイル構成。バックエンドなし・API キー不要。

![screenshot](docs/screenshot.png)

## 機能

- **2画面モード**: PC（901px 超）では自動的に左右2ペインに分割され、別銘柄のチャート・板・
  注文欄を同時に扱える。最上段にウォレット接続・画面切替・設定の共有コントロールと
  共通ティッカー、**最下段にポジション・オープンオーダーの共有欄**（両ペイン分を重複表示せず
  画面下に1つだけ全幅表示。クローズ・変更・取消もここから）。ペイン内の注文フォームは
  コンパクト表示で板・約定に高さを譲る。「1画面」ボタンで単独表示に切替（選択は記憶される）。
  スマホは常に1画面
- サブティッカー: HIP-3 xyz DEX の WTIOIL / GOLD / Nasdaq100 / MU / SPCX の mark・24h% を常時表示
- ローソク足チャート（15m / 1h / 4h / 1d / 1w / 1M、デフォルト日足。出来高付き、最新足に自動追従）— lightweight-charts
  - 指標: 移動平均線（20/50）・ボリンジャーバンド（20, 2σ）。トグルで表示切替
  - **自動トレンドチャネル（TL）・チャネル×BB シグナル・急落警報** — 詳細は後述の「自動テクニカル分析」
- リアルタイム板（累積深度バー・スプレッド表示）/ 約定履歴 / Mark・Funding・OI 等のステータス
- 銘柄切替: 通常 PERP に加え **HIP-3 builder DEX の株・商品・指数 perp** も出来高順で一覧
  （表示は公式 UI と同じ表示名 + DEX 名）。検索欄に入力するとプルダウンが絞り込まれ、
  Enter で先頭候補に切替（例: 「ET」→ ETH-PERP）
- ウォレット接続（MetaMask SDK）: ブラウザ拡張、またはQRをスマホの MetaMask で承認
  - 接続後、資産・ポジション・オープン注文を表示（2画面時は両ペインに反映）
  - アドレス手入力のウォッチモードあり（`?user=0x…` でも可）
- **発注**（指値 GTC / 成行 IOC / Reduce Only / 注文の修正・キャンセル / ポジションクローズ）
  - **レバレッジ変更**（倍率 + Cross/Isolated 切替、現在値と銘柄ごとの上限を表示）
  - API ウォレット方式: MetaMask の署名で agent 鍵を承認（approveAgent）し、以後の注文は
    ブラウザ内の agent 鍵で EIP-712 署名して `POST /exchange`。メイン秘密鍵はブラウザに置かない
  - agent 鍵は localStorage に平文保存される。共有マシンでは使わないこと
  - builder DEX 銘柄は現状表示のみ（発注・レバレッジ変更は非対応）
- API 接続先の切替（⚙）: 公式 Mainnet / 公式 Testnet / 代替プールのプリセット
  （api-ui.hyperliquid.xyz・api2.hyperliquid.xyz — レートリミット回避のフォールバック）/
  カスタム URL（キー付きサードパーティ API サーバー等）
- 表示言語の切替（最上段右端の JP/EN ボタン。既定は日本語、選択は記憶される）

## 自動テクニカル分析

手描きしていた平行線（トレンドチャネル）の自動化から始まり、
「**2〜4ヶ月の緩い上昇/横ばいチャネル → 天井過熱 → 床破断 → 急落**」という
再帰パターンの先取りを目的とした一連の機能。すべて日足を前提に設計している
（チャネル描画自体は全時間足で動く）。

### 自動トレンドチャネル（TL トグル・黄破線 = 検出区間 / teal 破線 = 延長）

各区間に「終値の回帰直線 ± 高値/安値へのオフセット」の平行線ペアを引く。
上下限は全バーを包む最大値ではなく**片側4%のバーのはみ出しを許容する分位点**
（極端なヒゲ数本でチャネル幅が水増しされない）。検出は3層:

1. **長期（トップダウン分割）** — 全300本を「チャネル幅 ≤ 平均価格の22%で一貫」に
   なるまで再帰2分割（分割点は子チャネル幅の最大値が最小になる位置を全候補走査）。
   その後、①隣接区間の再統合、②境界±20本の幅コスト最適化、③境界を近傍の
   天井/大底へスナップ、④一貫性を失った区間の再分割、を順に適用。
   ピボットに依存しないため**数ヶ月のレンジ帯が一組の平行線にまとまり**、
   境界は人間が引くのと同じく転換点（天井・大底）に乗る。
2. **短期/中期（ピボット）** — スイング高値/安値（±2本・±4本の最値）の H/L 交互列から
   HH/HL・LH/LL の反転点で区間化し、折れ点での再帰分割で急落ともみ合いの融合を分離。
   描画するのは「右端に届く現況チャネル」か「直近90日以内の急変
   （重なる長期バンドを12本未満で横断する傾き差 = 急落/急騰）」だけ。
   過去の細かいレッグは大局把握のノイズになるため描かない。
3. **直近レッグ** — トレンド反転が未確定でも、直近30本の最安値/最高値→現在を
   常時チャネル候補にする（新しいレッグは約8本 ≒ 日足1週間強で自動出現）。

区間の外側へは「バーがチャネル内（幅の20%まで許容）に収まる限り」左右へ延長し
（上限 = 自身の区間長）、右端まで届いた緩やかなチャネルは未来へ8本延長する。
本体（黄）と延長（teal）の色分けで、どこからが外挿かが分かる。
チャネルは固定オブジェクトではなく**足の確定ごとに全履歴から再導出**されるため、
時間の経過で自動的に追加・延長・引き直しされる。

### チャネル×BB 交差シグナル（▼▲マーカー）

- **▼**（足の上・売り色）: BB 下バンドがチャネル下線を上から下へ突き抜けた足（床破断）
- **▲**（足の下・買い色）: BB 上バンドがチャネル上線を下から上へ突き抜けた足（天井過熱/ブレイク）

判定はチャネルの実区間+フィット確認済みの延長範囲。TL と BB の両トグル ON で表示。
2026-06-01 の急落開始（緩上昇チャネル下線×BB 下バンドの交差）を再現する。

### パターン監視バッジと総合急落警報

チャート左上のバッジが、右端まで生きている長期チャネルの状態を常時表示する
（日足のみ）: `パターン監視: 6/4〜 39日目 · 天井過熱 ▲ 15日前 · 弱気局面 · 実需5分: 現物売/perp中立`

| 表示 | 意味 |
|---|---|
| 監視中（灰） | 2〜4ヶ月級の緩い上昇/横ばいチャネルが右端で進行中 |
| 天井過熱（琥珀） | そのチャネル自身の ▲ が15本以内 — 床破断を警戒する局面 |
| 床破断（赤） | ▼ が5本以内 — 過去の急落はここから始まった |
| 弱気/強気局面 | 終値と SMA200 の位置。破断シグナルは弱気局面でのみ信頼する（下記） |
| 実需5分 | crypto_analysis の realtime_demand 連携（任意・下記） |

さらに **局面 × パターン段階 × 実需の合議**で急落警報を出す:

| レベル | 条件 | 動作 |
|---|---|---|
| 🚨 3 急落警報 | 弱気局面 + 床破断 + 現物の買い支えなし | 赤点滅バナー + ブラウザ通知 + ビープ（日付×レベルで重複抑止） |
| ⚠ 2 急落警戒 | 破断だが現物買い（騙し注意）/ 弱気局面の天井過熱 | 琥珀バナー、タブタイトルに ⚠ |
| 1 | 弱気局面で監視中 | バッジのみ |

### 14年バックテスト（2026-07-13 実施）

Bitstamp BTC/USD 日足 2012〜2026（5,307本）を、この app.js 実物にウォークフォワードで
流して検証（各日、その日までの300本だけで再計算 — 未来のデータは見えない）。

- 床破断シグナル（全65回）から30日以内に **-18% 超の大暴落**が来た率:
  **弱気局面（終値<SMA200）32% / 強気局面 5%** — 破断の価値は弱気局面の尾部に集中
- 各サイクル下落局面の看板暴落をほぼ全て事前捕捉:
  2014-08/-11（-14%/-19%）・**2018-11-16（-42%、前日）**・**2020-02-26（COVID -45%、2週間前）**・
  2021-11（天井直後）・**2022-04/06/08（-28%/-33%/-22%）**・2025-10・2026-01×2・2026-05
- 限界: 弱気局面でも破断の約半数は -5% 未満の不発（**単独では張れない — 実需併用が前提**）。
  捕まえるのは「成熟した持ち合いの床割れ」型のみで、ベア中盤の連鎖階段は対象外。

詳細・再現手順: crypto_analysis プロジェクトの `notes/btc-channel-break-backtest.md`（ローカル）

### 実需フィード連携（任意）

同一ホストの port 8765 で別プロジェクト crypto_analysis の realtime_demand サーバー
（Binance spot/perp + Coinbase spot のテイカーフロー配信）が動いていれば自動接続し、
直近5分の現物/perp フロー方向をバッジに追記、警報レベルの判定に使う。
床破断時に現物が買い優勢なら「騙しの可能性」（2026-04-01 に ▼ を実需の買いが覆した教訓）、
現物も売りなら「信頼度高」。サーバー停止時は静かにテクニカルのみ表示に戻る。
LAN から使う場合はサーバーを `--host 0.0.0.0` で起動しておく。
localStorage `hlt-demand` に URL を入れると接続先を上書き、`off` で無効化。

### 主要パラメータ（app.js 冒頭の TL_* 定数）

| 定数 | 既定 | 意味 |
|---|---|---|
| `TL_OUTLIER` | 0.04 | チャネル上下限が無視してよいバーの割合（片側） |
| `TL_COHERENT_FRAC` | 0.22 | 「一貫したチャネル」とみなす幅の上限（平均価格比） |
| `TL_SPLIT_GAIN` | 0.75 | 再帰分割の採用条件（子幅がこの比率未満に縮む場合のみ） |
| `TL_MIN_SUB` / `TL_COARSE_MIN` | 8 / 10 | 分割子・長期チャネルの最小本数 |
| `TL_RECENT_WIN` | 30 | 直近レッグの起点を探す窓 |
| `TL_FAST_BARS` | 12 | 「急変」とみなす傾き差（長期バンドを何本で横断するか） |
| `TL_RECENT_CRASH` | 90 | 急変チャネルを描画する新しさの上限 |
| `TL_EXTEND` / `TL_FIT_TOL` | 8 / 0.2 | 未来延長の本数 / 延長続行の許容はみ出し |
| `TL_MAX_CHANNELS` | 16 | 描画上限 |

## 使い方

```sh
./build-sdk.sh   # 初回のみ: metamask-sdk.bundle.js を生成（npm が必要）
./run.sh [port]  # デフォルト 8010
```

ブラウザで `http://<ホスト>:8010/` を開く。PC では自動的に2画面（`duo.html`）になり、
スマホでは1画面で表示される。

MetaMask SDK のバンドルは ConsenSys の独自ライセンス（再配布不可）のため
リポジトリに含めていません。`build-sdk.sh` が npm から取得して生成します。
